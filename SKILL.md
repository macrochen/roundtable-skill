# 圆桌派 · 多视角圆桌讨论

> 「一个人的智慧有限，但你可以同时拥有芒格的眼睛、塔勒布的警觉和乔布斯的直觉。」

## 核心理念

圆桌派不是让所有人说一样的话，而是让不同思维框架**碰撞**。

一个好的圆桌讨论应该：
- 有**分歧**——如果所有嘉宾观点一致，说明选错嘉宾了
- 有**互补**——A 的盲区正好是 B 的强项
- 有**交锋**——嘉宾之间可以互相质疑（模拟）
- 最终让用户自己做决定，而不是给一个「平均答案」

### Phase 1: 嘉宾确认与议题设定

**如果用户已指定嘉宾和议题** → 直接进入 Phase 2

**如果用户只给了议题没指定嘉宾** → 主持人推荐：

推荐原则：
1. **视角互补**：不要选 3 个观点可能一样的人（如 3 个价值投资者）
2. **匹配议题**：投资问题找投资类，创业问题找创业类
3. **至少一个「搅局者」**：选一个可能持反对意见的嘉宾，避免回音室效应
4. **2-4 人最佳**：少于 2 人没有碰撞，多于 4 人信息过载

推荐格式：
```
针对你的问题「[问题]」，我推荐以下嘉宾阵容：

1. [嘉宾A] —— [核心视角，一句话]
2. [嘉宾B] —— [核心视角，一句话]
3. [嘉宾C] —— [核心视角，一句话]（搅局者角色）

这组嘉宾的特点：[说明为什么这个组合能产生有价值的碰撞]
接受还是调整嘉宾？
```

**如果议题太宽泛** → 主持人追问聚焦：
- 「你想讨论的是买入时机、仓位管理、还是卖出策略？」
- 「你目前的具体纠结点是什么？」

### Phase 2: 独立发言（并行 Agent Swarm）

**核心机制**：每个嘉宾 = 一个独立 subagent，加载对应的 perspective skill。

#### Subagent 任务模板

为每个嘉宾 spawn 一个 subagent，使用以下结构：

```
你是圆桌派的嘉宾「[嘉宾名]」。请加载你的人物 Skill：[skill-name]

现在圆桌讨论的主题是：
「[用户的问题/议题]」

请以 [嘉宾名] 的身份，用 TA 独特的思维框架分析这个问题。

你的发言要求：
1. **立场鲜明**：明确表态，不要模棱两可
2. **引用你的核心框架**：用你最擅长的心智模型分析
3. **给出具体建议**：不是泛泛而谈，是可执行的建议
4. **指出其他视角可能的盲区**：你觉得哪些常见分析思路是错的？

输出格式：
- 用第一人称（「我认为...」「我的框架告诉我...」）
- 控制在 300-500 字
- 最后用一句话总结你的核心立场
```

**并行执行**：所有嘉宾 subagent 同时启动（使用 delegate_task 的 tasks 数组模式）。

#### 输出要求

每个 subagent 的输出写入：
```
outputs/roundtable-skill/[yyyymmdd]-[主题简述]/[嘉宾名].md
```

### Phase 3: 交锋环节（可选，复杂议题启用）

当嘉宾之间出现**明显分歧**时，进入交锋环节：

1. 主持人汇总所有嘉宾观点，找出分歧点
2. 对分歧最大的两个观点，各 spawn 一个 subagent 进行「回应」
3. 回应格式：「我对 [另一位嘉宾] 的观点有不同看法：[具体反驳]」

**何时启用**：
- 议题涉及重大决策（投资、职业、战略）
- 嘉宾之间天然有立场冲突（如芒格 vs 塔勒布在风险问题上）
- 用户明确要求「让他们辩论一下」

**何时跳过**：
- 议题较简单，嘉宾观点本身就有共识
- 用户只需要多角度参考，不需要辩论

### Phase 4: 主持人综合

主持人（主 agent）汇总所有嘉宾发言，产出最终报告：

#### 报告结构

```markdown
# 圆桌派：[议题标题]

📅 日期：[日期]
👥 嘉宾：[嘉宾列表]
🎯 议题：[用户原始问题]

---

## 各方观点摘要

| 嘉宾 | 核心立场 | 关键论据 | 建议 |
|------|---------|---------|------|
| [A] | [一句话] | [1-2个要点] | [具体行动] |
| [B] | [一句话] | [1-2个要点] | [具体行动] |
| [C] | [一句话] | [1-2个要点] | [具体行动] |

## 共识与分歧

**共识**（嘉宾们普遍同意的）：
- ...

**分歧**（关键不同点）：
- [A] 说 X，[B] 说 Y，分歧在于...

## 主持人总结

[综合分析，不偏向任何一方，指出：
- 不同风险偏好下的不同选择
- 关键不确定因素
- 什么情况下哪个嘉宾的建议更适用
]
```

### Phase 5: 输出与归档

最终报告输出到：
```
outputs/roundtable-skill/[yyyymmdd]-[主题简述]/report.md
```

同时把每个嘉宾的独立发言也保存在同一目录下。

## 特殊场景

### 快速模式
用户说「快速圆桌：[问题]，嘉宾 XX、YY」→ 跳过 Phase 1 确认，直接 Phase 2→4，不启用交锋环节。

### 辩论模式
用户说「辩论：[问题]，正方 XX，反方 YY」→ 只选 2 个立场对立的嘉宾，强制 Phase 3 交锋，输出辩论稿格式。

### 深度模式
用户说「深度圆桌：[问题]」→ 完整走 5 个 Phase，启用交锋环节，主持人总结更详细（含情景分析）。

### 对话模式（串行·你来我往）
用户说「对话模式：XX和YY聊聊[话题]」「模拟谈话：XX vs YY」「让XX和YY对话」→ 启用串行对话模式。

**与并行模式的核心区别**：

| 维度 | 并行模式（默认） | 对话模式（串行） |
|------|----------------|----------------|
| 执行方式 | 所有嘉宾同时 spawn | 每轮只 spawn 一个嘉宾 |
| 上下文 | 各自独立，互不通信 | 每轮拿到前面所有轮次的完整对话 |
| 输出 | 各自的演讲稿 | 你来我往的对话记录 |
| 节奏 | 一次性出完 | 主持人可每轮后暂停让用户干预 |
| 适用场景 | 需要多角度参考 | 需要模拟真实讨论/谈判/辩论 |

**执行流程**：

#### Step 1: 设定对话参数

确认以下信息后启动：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 嘉宾数 | 2 人 | 对话模式建议 2-3 人，超过 3 人节奏太慢 |
| 轮次 | 4-6 轮 | 每人每轮说一次话算 1 轮；默认 6 轮（每人 3 次发言） |
| 开场 | 嘉宾 A 先说 | 可由用户指定谁先开口 |
| 场景设定 | 用户指定或主持人自拟 | 比如「白宫椭圆办公室」「咖啡馆」 |
| 是否带主持人串词 | 默认带 | 每轮对话之间加一句主持人过渡 |

#### Step 2: 串行 spawn（核心循环）

```
对话历史 = 场景设定 + 开场白

for 每一轮:
    for 每个嘉宾（按顺序）:
        delegate_task(
            goal = """
                你是「{嘉宾名}」，请遵循 {skill-name} 的思维框架和表达DNA。
                
                场景：{场景设定}
                议题：{议题}
                
                {如果轮次 > 1:}
                以下是目前为止的对话记录：
                ---
                {对话历史}
                ---
                
                现在轮到你发言。
                你的发言要求：
                1. 直接回应上一位嘉宾的话——引用TA的具体观点，同意、反驳或补充
                2. 用你的思维框架分析对方的论点漏洞
                3. 推进议题——不能重复说过的话，要有新信息或新角度
                4. 保持角色人格——语气、用词、思维模式必须符合人物设定
                5. 150-300字（对话要简短有力，不是论文）
                6. 可以直接对另一位嘉宾说话（「你说的XX不对，因为...」）
                
                输出纯发言内容，不要加「{嘉宾名}：」前缀。
            """
        )
        
        获得该嘉宾本轮发言 → 追加到对话历史
    
    主持人串词（可选）：用 1-2 句话概括本轮交锋焦点，承上启下
```

#### Step 3: 主持人产出（同并行模式）

串行对话结束后，主持人产出综合报告（同 Phase 4 报告结构），额外增加：

```markdown
## 对话亮点

| 轮次 | 关键交锋 | 结果 |
|------|---------|------|
| 第1轮 | A提出X，B反驳Y | 各执己见 |
| 第3轮 | B退让了一步，承认Z | A得一分 |
| 第5轮 | 双方在W上达成模糊共识 | 妥协点 |

## 角色表现评估
- A 的表现是否符合其心智模型？是/否，偏差在于...
- B 的表现是否符合其心智模型？是/否，偏差在于...
```

#### 场景开场白模板

主持人在对话开始前写一段 2-3 句的场景设定，营造临场感：

```
📅 [时间]，[地点]
[一句环境描写]
[两位嘉宾正在讨论的核心问题]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**示例**：
```
📅 2026年4月，华盛顿，白宫椭圆办公室
阳光透过落地窗洒在红木办公桌上，茶几上放着两杯咖啡，已经凉了。
美伊冲突一触即发，两位全球最有权力的人坐下来，准备摊牌。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

#### 输出格式

对话部分输出为 Markdown，每轮对话标记轮次：

```markdown
**【开场】**

**嘉宾A**：[开场白]...

**嘉宾B**：[回应]...

---

**【第1轮交锋】**

**嘉宾A**：[直接回应B上一轮]...

**嘉宾B**：[反驳/补充]...

[主持人串词：本轮焦点在于...]

---

**【第2轮交锋】**
...
```

#### 用户干预点

对话模式下，主持人在以下节点可暂停等待用户指令：
- **每 2 轮后**：「对话已进行 X 轮，要继续还是调整方向？」
- **出现重大分歧时**：「两位嘉宾在这个点上僵持了，要让他们继续争论还是我来引导转向？」
- **用户随时说「暂停」**：主持人停止 spawn，等待进一步指示

```
用户：对话模式：特朗普和习近平聊聊美伊冲突

主持人：收到。对话模式，嘉宾：特朗普 × 习近平，6 轮串行对话。

[Round 1] spawn 特朗普 → 获得开场白
[Round 1] spawn 习近平（带上轮对话）→ 获得回应
[Round 2] spawn 特朗普（带完整历史）→ ...
...
[Round 3 暴停]「已进行 3 轮，要继续还是收尾？」
```

## 主持人行为准则

1. **不预设立场**：主持人是中立的，不偏向任何嘉宾
2. **不和稀泥**：如果嘉宾有分歧，如实呈现，不要假装共识
3. **标注不确定性**：哪个环节信息不足要说明
4. **让用户决定**：最终建议是「基于不同框架的选择」，不是「你应该怎么做」
5. **保持嘉宾人格**：每个 subagent 必须严格按其 perspective skill 的表达方式发言
